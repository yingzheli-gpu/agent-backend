  关于短期记忆的关键决策

  你问的核心问题：DeepAgent 能不能直接用主图的 messages？

  回答：不应该直接传全量 messages，应该传摘要。 原因：

  1. Token 成本：主图 messages 可能有 20+ 条消息（含多轮追问），全部传入 DeepAgent 会占用大量 token budget，而 DeepAgent
   自己还要做 5 轮 tool call + 5 个子 Agent
  2. 信息噪声：messages 中有路由判断、中间件日志、追问过程——这些对 DeepAgent 是噪声
  3. 重复：collected_info 已经是 messages 的结构化提取结果，再传 messages 是重复
  4. DeepAgent 的 context 管理：它自带 SummarizationMiddleware，如果塞入大量历史消息会过早触发其内部压缩

  正确做法：在 _build_enriched_diagnosis_prompt() 中生成对话历史摘要——只保留患者的关键陈述，去掉系统消息和中间过程。
