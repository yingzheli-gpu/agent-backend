# 复杂辨证工作流程

## 概述

此技能提供复杂病例的完整辨证分析流程，包括数据收集、专家咨询和综合决策三个阶段。

## 何时使用此技能

- 患者症状复杂，涉及多个脏腑
- 症状之间存在矛盾（如寒热错杂）
- 需要深度分析病因病机
- 需要多专家协同诊断

## 辨证步骤

### 步骤 1：症状分析

使用 `write_todos` 工具创建任务清单：
- 分析主症与兼症
- 识别症状矛盾点
- 判断涉及的脏腑系统

### 步骤 2：并行数据查询

同时调用 4 个数据工具：

1. **kg_syndrome_search** - 查询知识图谱证型
   ```
   kg_syndrome_search(symptoms=["头痛", "胸闷", "失眠"])
   ```

2. **case_vector_search** - 检索相似医案
   ```
   case_vector_search(query="头痛 胸闷 失眠 腰膝酸软")
   ```

3. **classics_search** - 检索古籍论述
   ```
   classics_search(keywords=["头痛", "肝郁"])
   ```

4. **web_search** - 搜索最新研究（可选）
   ```
   web_search(query="头痛 中西医结合治疗")
   ```

### 步骤 3：并行专家咨询

将数据提供给 5 位专家子 Agent：

1. **differential_diagnosis_expert** - 鉴别诊断
   - 区分相似证型
   - 分析寒热虚实真假
   - 排除误诊可能

2. **treatment_principle_expert** - 治则治法
   - 确定治本/治标策略
   - 制定扶正祛邪方案
   - 因时因地因人制宜

3. **prescription_recommendation_expert** - 方药推荐
   - 推荐主方及加减
   - 提供食疗方案
   - 建议穴位调理

4. **prognosis_evaluation_expert** - 预后评估
   - 评估正气盛衰
   - 判断疾病传变风险
   - 预测病情趋势

5. **verification_expert** - 质疑验证
   - 验证逻辑完整性
   - 检查证据充分性
   - 确保安全性

### 步骤 4：综合决策

基于所有专家意见：
1. 确定主证与兼证
2. 综合病因病机分析
3. 确定治法治则
4. 计算置信度
5. 如有分歧，说明理由

### 步骤 5：验证迭代

使用 validation-criteria 技能中的标准：
- 与相似医案对比
- 检查理论一致性
- 评估置信度
- 如不合理，返回步骤 4 重新分析（最多 3 次）

## 注意事项

- 始终先完成数据查询，再咨询专家
- 如专家意见有冲突，需仔细权衡
- 最终结论必须有充分证据支持
- 本诊断仅供参考，具体诊疗请咨询专业中医师
