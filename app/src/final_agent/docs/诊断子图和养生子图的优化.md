# 诊断子图和养生子图上下文闭环优化方案

## 1. 问题分析

### 1.1 核心问题：上下文隔离

当前 MainGraph 的中间件（Memory、Learning、Focus）在主图层面已完成，但诊断子图和养生子图存在严重的上下文隔离问题：

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| DiagnoseInputState 只传递 5 个字段 | memory_context、learning、user_id 等无法进入子图 | 高 |
| DeepAgent 完全隔离 | 每次创建新 MemorySaver + 新 thread_id，无主图上下文 | 高 |
| `complex/__init__.py` 导入已删除模块 | `from .deep_search_agent import` 应为 `_custom` 版本 | 阻塞 |
| `router.py` 将复杂问题降级为 moderate | 复杂路径永远无法触发 | 高 |
| 简单/中等诊断不使用 memory_context | 历史记忆和学习经验被浪费 | 中 |
| `builder.py` _import_handlers 指向 super_agent | final_agent 主图导入了错误模块的 handlers | 阻塞 |
| `complex_diagnosis.py` 导入 super_agent 的 states | 应该导入 final_agent 自己的 states | 高 |
| 诊断子图单例模式导致上下文工具无法动态绑定 | 单例 `_diagnose_graph` 无法感知每次请求的不同 context | 中 |
| WellnessInputState 缺少上下文字段 | 养生子图同样无法获取记忆和学习经验 | 中 |
| moderate 的 Map-Reduce 子图 ModerateState 不含上下文 | synthesize_diagnosis 无法注入学习经验 | 中 |
| `call_wellness_subgraph` 兼容旧 handler 签名 | handler 缺少 config 参数，且不传递上下文 | 中 |

### 1.2 数据流断裂点

```
MainGraph MainState (完整上下文)
  ├── user_id ✓
  ├── conversation_id ✓
  ├── memory_context ✓ (MemoryMiddleware 加载)
  ├── cross_thread_learning ✓ (LearningMiddleware 加载)
  ├── thread_learning_context ✓ (LearningMiddleware 加载)
  ├── user_profile (在 memory_context.factual 内)
  └── messages ✓ (FocusContext 裁剪过)
        │
        ▼ handle_diagnose_query() ← 只传递 5 个字段！
        │
  DiagnoseInputState (残缺上下文)
  ├── query ✓
  ├── messages ✓
  ├── user_profile ✓
  ├── llm_config ✓
  └── extracted_entities ✓ (NotRequired)
  ├── user_id ✗ ← 缺失
  ├── conversation_id ✗ ← 缺失
  ├── memory_context ✗ ← 缺失
  ├── cross_thread_learning ✗ ← 缺失
  ├── thread_learning_context ✗ ← 缺失
        │
        ▼ complex_diagnosis → DeepAgent ← 完全隔离！
        │
  DeepAgent (独立上下文)
  ├── 新 MemorySaver (非主图 checkpointer)
  ├── 新 thread_id = f"diagnosis_{user_id}_{timestamp}" (无法关联)
  ├── 只传 1 条 {"role":"user","content": prompt} (丢弃对话历史)
  └── 无 memory/learning 工具
```

### 1.3 关键发现：messages 应直接传入 DeepAgent

主图已有完整的消息管理链：
1. `FocusContextMiddleware (context.py)`: 总结-裁剪策略，主图保留最近 10 条
2. `FocusContextEngine (focus_engine.py)`: Sawtooth 压缩
3. `TCMSummarizer`: 旧消息压缩成 `[对话历史摘要]` 的 SystemMessage

因此当 messages 到达诊断子图时，已经是裁剪过的精简版本。

三个诊断路由应如何使用 messages：

| 路由 | 当前用法 | 应该怎么用 |
|------|----------|-----------|
| simple | 不用 messages，只用 collected_info | 可以直接把 messages 传给 LLM 作为对话上下文 |
| moderate | 不用 messages，synthesize 只看 RAG 结果 | synthesize 阶段附带 messages 给 LLM 完整上下文 |
| complex | 完全丢弃，DeepAgent 从零开始 | 直接传入 messages 作为 DeepAgent 的初始对话历史 |

### Phase 3: 增强 simple_diagnosis

**目标**: 简单诊断利用历史记忆和学习经验增强 prompt

**文件**: `components/diagnose/nodes/simple/simple_diagnosis.py`

#### 3.1 构建上下文注入辅助函数

在文件中新增一个辅助函数：

```python
def _build_context_injection(state: DiagnoseOverallState) -> str:
    """从 state 中提取记忆和学习上下文，构建 prompt 注入段"""
    memory_ctx = state.get("memory_context") or {}
    learning_ctx = state.get("cross_thread_learning") or {}

    sections = []

    # 1. 历史诊疗记忆摘要（取前 3 条）
    episodic = memory_ctx.get("episodic", {}).get("summaries", [])[:3]
    if episodic:
        summaries = "\n".join(
            f"- {m.get('memory', m.get('text', ''))}" for m in episodic if isinstance(m, dict)
        )
        if summaries.strip():
            sections.append(f"【历史诊疗记忆】\n{summaries}")

    # 2. 相关语义实体
    entities = memory_ctx.get("semantic", {}).get("entities", [])
    if entities:
        entity_str = "、".join(str(e) for e in entities[:10])
        sections.append(f"【相关知识实体】\n{entity_str}")

    # 3. 学习经验：鉴别规则
    rules = learning_ctx.get("discriminating_rules", [])
    if rules:
        rules_str = "\n".join(
            f"- {r.get('syndrome_a', '')} vs {r.get('syndrome_b', '')}: {r.get('rule', '')}"
            for r in rules[:5] if isinstance(r, dict)
        )
        if rules_str.strip():
            sections.append(f"【鉴别经验】\n{rules_str}")

    # 4. 误诊模式提醒
    patterns = learning_ctx.get("misdiagnosis_patterns", [])
    if patterns:
        patterns_str = "\n".join(
            f"- 注意区分 {p.get('wrong_syndrome', '')} 和 {p.get('correct_syndrome', '')}: {p.get('prevention_rule', '')}"
            for p in patterns[:3] if isinstance(p, dict)
        )
        if patterns_str.strip():
            sections.append(f"【历史误诊提醒】\n{patterns_str}")

    return "\n\n".join(sections) if sections else ""
```

**注意**:
- `memory_context` 的实际结构是 `MemoryContext` TypedDict，episodic 数据在 `memory_context["episodic"]["summaries"]` 而非 `memory_context["episodic_memories"]`。但 `_build_memory_context()` (memory.py:269) 同时设置了 `episodic_summaries` 和 `episodic_memories` 顶层字段，所以需要兼容两种路径。
- `cross_thread_learning` 的结构是 `CrossThreadLearningContext` TypedDict，其中 `discriminating_rules` 是 `List[DiscriminatingRuleRecord]`（每条是 dict 包含 syndrome_a, syndrome_b, rule 等字段），不是纯字符串。

```python
    patterns = learning_ctx.get("misdiagnosis_patterns", [])
    if patterns:
        patterns_str = "\n".join(
            f"- {p.get('pattern_name', '')}: {p.get('prevention_rule', '')}"
            for p in patterns[:3] if isinstance(p, dict)
        )
        if patterns_str.strip():
            sections.append(f"【误诊模式提醒】\n{patterns_str}")

    # 有效策略
    strategies = learning_ctx.get("effective_strategies", [])
    if strategies:
        strat_str = "\n".join(
            f"- {s.get('strategy_name', '')}: {s.get('description', '')}"
            for s in strategies[:3] if isinstance(s, dict)
        )
        if strat_str.strip():
            sections.append(f"【有效诊断策略】\n{strat_str}")

    # 历史病例记忆
    episodic = memory_ctx.get("episodic", {}).get("summaries", [])[:3]
    if episodic:
        summaries = "\n".join(
            f"- {m.get('memory', m.get('text', ''))}" for m in episodic if isinstance(m, dict)
        )
        if summaries.strip():
            sections.append(f"【相似病例记忆】\n{summaries}")

    return "\n\n".join(sections) if sections else ""
```

### Phase 5b: 创建 context_tools.py

**目标**: 为 DeepAgent 提供记忆搜索和学习规则查询工具

**新文件**: `components/diagnose/nodes/complex/tools/context_tools.py`

**设计决策**: 使用**闭包工厂模式**，在 `complex_diagnosis.py` 创建 Agent 时，将当前请求的 memory_context 和 learning_context 绑定到工具函数中，避免全局状态污染。

```python
"""
上下文查询工具（闭包工厂模式）

在 DeepAgent 创建时绑定当前请求的 memory_context 和 learning_context，
使 Agent 可以通过工具调用查询历史记忆和学习规则。
"""

from typing import Dict, Any, List, Optional
from langchain_core.tools import tool


def create_context_tools(
    memory_context: Optional[Dict[str, Any]] = None,
    learning_context: Optional[Dict[str, Any]] = None,
) -> list:
    """
    工厂函数：创建绑定了当前请求上下文的工具列表

    Args:
        memory_context: MemoryMiddleware 加载的长期记忆
        learning_context: LearningMiddleware 加载的跨线程学习经验

    Returns:
        绑定了上下文的工具列表
    """
    tools = []

    if memory_context:
        @tool
        def memory_search(query: str) -> str:
            """搜索患者历史诊疗记忆，查找相似症状和既往诊断结果。

            当需要了解患者是否有类似的历史症状、既往诊断、或过去的治疗方案时使用。

            Args:
                query: 症状关键词或诊断相关查询
            Returns:
                相关历史记忆摘要
            """
            results = []

            # 搜索 episodic memories
            episodic = memory_context.get("episodic", {}).get("summaries", [])
            if not episodic:
                episodic = memory_context.get("episodic_summaries", []) or memory_context.get("episodic_memories", [])

            for m in episodic:
                if not isinstance(m, dict):
                    continue
                text = m.get("memory", m.get("text", ""))
                # 简单关键词匹配（Mem0 已做向量检索，这里做二次过滤）
                if any(keyword in text for keyword in query.split()):
                    score = m.get("score", "N/A")
                    results.append(f"- {text} (相关度: {score})")

            # 搜索 semantic 实体
            entities = memory_context.get("semantic", {}).get("entities", [])
            matching_entities = [e for e in entities if any(k in str(e) for k in query.split())]
            if matching_entities:
                results.append(f"- 相关实体: {'、'.join(str(e) for e in matching_entities[:5])}")

            # 搜索 semantic 关系
            relations = memory_context.get("semantic", {}).get("relations", [])
            matching_relations = [
                r for r in relations
                if isinstance(r, dict) and any(k in str(r) for k in query.split())
            ]
            if matching_relations:
                for r in matching_relations[:3]:
                    results.append(f"- 关系: {r.get('subject','')} → {r.get('predicate','')} → {r.get('object','')}")

            if not results:
                return f"未找到与「{query}」相关的历史记忆"

            return "\n".join(results[:10])

        tools.append(memory_search)

    if learning_context:
        @tool
        def learning_rules_query(syndrome_or_symptom: str) -> str:
            """查询跨线程学习的鉴别规则、历史误诊模式和有效诊断策略。

            当需要区分相似证型、避免误诊、或参考有效诊断策略时使用。

            Args:
                syndrome_or_symptom: 证型名称或症状组合
            Returns:
                相关鉴别规则、误诊提醒和有效策略
            """
            results = []
            query_lower = syndrome_or_symptom.lower()

            # 鉴别规则
            rules = learning_context.get("discriminating_rules", [])
            for r in rules:
                if not isinstance(r, dict):
                    continue
                if (query_lower in str(r.get("syndrome_a", "")).lower()
                    or query_lower in str(r.get("syndrome_b", "")).lower()
                    or any(query_lower in str(s).lower() for s in r.get("symptoms", []))):
                    results.append(
                        f"【鉴别规则】{r.get('syndrome_a','')} vs {r.get('syndrome_b','')}: {r.get('rule','')}"
                    )

            # 误诊模式
            patterns = learning_context.get("misdiagnosis_patterns", [])
            for p in patterns:
                if not isinstance(p, dict):
                    continue
                if (query_lower in str(p.get("wrong_syndrome", "")).lower()
                    or query_lower in str(p.get("correct_syndrome", "")).lower()
                    or any(query_lower in str(s).lower() for s in p.get("missed_symptoms", []))):
                    results.append(
                        f"【误诊提醒】{p.get('pattern_name','')}: "
                        f"易误判为{p.get('wrong_syndrome','')}, 实为{p.get('correct_syndrome','')}. "
                        f"预防: {p.get('prevention_rule','')}"
                    )

            # 有效策略
            strategies = learning_context.get("effective_strategies", [])
            for s in strategies:
                if not isinstance(s, dict):
                    continue
                if (any(query_lower in str(sym).lower() for sym in s.get("applicable_symptoms", []))
                    or any(query_lower in str(syn).lower() for syn in s.get("applicable_syndromes", []))):
                    results.append(
                        f"【有效策略】{s.get('strategy_name','')}: {s.get('description','')}"
                    )

            if not results:
                return f"未找到与「{syndrome_or_symptom}」相关的鉴别规则或诊断策略"

            return "\n".join(results[:10])

        tools.append(learning_rules_query)

    return tools
```

## 8. 风险与注意事项

1. **State 兼容性**: 新增字段全部使用 `NotRequired`，不传时不报错，不影响已有调用路径
2. **性能**: context_tools 使用闭包绑定而非每次重新加载，开销极小
3. **循环依赖**: context_tools.py 不直接 import middleware，通过工厂函数注入
4. **回退机制**: 所有上下文注入都有空值保护，memory/learning 不可用时不影响基本诊断
5. **asyncio.run 冲突**: LearningMiddleware.after_model 内部的 `asyncio.run()` 在 LangGraph 异步节点中可能报错，需确认并可能改为 `await`
6. **诊断子图单例**: `_diagnose_graph` 是全局单例，图结构不变，但 context_tools 在每次调用时动态绑定（通过 `create_deep_search_agent_custom` 的参数），所以单例不影响上下文隔离
7. **TCMAgentState vs MainState**: `handle_diagnose_query` 的类型注解是 `TCMAgentState`（Pydantic model），但 final_agent 用 `MainState`（TypedDict），需统一为 dict 访问方式（`.get()` 而非属性访问）
8. **memory_context 结构差异**: `_build_memory_context()` 返回的字典同时有顶层 `episodic_summaries`/`episodic_memories` 和嵌套 `episodic.summaries`，代码中需兼容两种路径
