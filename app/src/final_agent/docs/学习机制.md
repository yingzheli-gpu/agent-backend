1.
系统概述

1.1
背景

本系统是面向C端用户的中医智能问诊系统，基于LangGraph多智能体架构，包含：
- 主图（Main
Graph）: 意图识别与路由
- 子图（Subgraphs）: 诊断、养生、草药咨询等专业子流程

1.2
核心目标

准确率是生命线。自学习机制的唯一目标是：

- 主图准确率: 意图识别准确率 > 90 %
- 子图准确率:
- 辨证诊断准确率 > 80 %
- 工具选择准确率 > 85 %
- 症状理解准确率 > 90 %
- 追问效率 > 0.7

1.3
学习架构

三层学习架构：
├── Feedback
Layer(反馈层)
│   └── 收集准确率相关的用户反馈
├── Reflection
Layer(反思层)
│   └── 分析错误原因，生成改进规则
└── Evolution
Layer(进化层)
└── 从案例中提取知识，提升准确率

2.
核心设计原则

2.1
准确率优先

- ✅ 所有学习活动必须直接服务于提升准确率
- ✅ 每个反馈、反思、进化都要有明确的准确率指标
- ❌ 不追求
"智能"，只追求
"准确"

2.2
主图与子图分离

┌──────────┬────────────────┬──────────────────────────────────┐
│   维度   │      主图      │               子图               │
├──────────┼────────────────┼──────────────────────────────────┤
│ 学习目标 │ 意图识别准确率 │ 诊断 / 工具 / 推理准确率             │
├──────────┼────────────────┼──────────────────────────────────┤
│ 学习内容 │ 意图纠偏规则   │ 辨证规则、工具选择规则、追问策略 │
├──────────┼────────────────┼──────────────────────────────────┤
│ 触发时机 │ 用户纠正意图时 │ 诊断错误、工具选错、追问低效时   │
└──────────┴────────────────┴──────────────────────────────────┘

2.3
单线程
vs
跨线程

┌────────────┬──────────────┬────────────────────┬────────────────────┐
│    类型    │    作用域    │      学习内容      │      生命周期      │
├────────────┼──────────────┼────────────────────┼────────────────────┤
│ 单线程学习 │ 当前对话     │ 即时纠偏、快速收敛 │ 对话结束后持久化   │
├────────────┼──────────────┼────────────────────┼────────────────────┤
│ 跨线程学习 │ 所有历史对话 │ 可泛化的模式和规则 │ 永久存储，持续积累 │
└────────────┴──────────────┴────────────────────┴────────────────────┘

2.4
可操作性原则

- ✅ 反思必须产生具体的判断规则（不是
"要注意"）
- ✅ 进化必须产生可应用的改进策略（不是
"建议优化"）
- ✅ 所有学习成果必须结构化，便于检索和应用

2.5
安全性原则

- ❌ 不自动修改系统架构
- ❌ 不自动切换模型
- ❌ 不自动修改核心提示词（需人工审核）
- ✅ 学习成果需要人工审核后才能应用到生产环境

学习维度划分

3.1
主图学习维度

3.1
.1
意图识别准确率

学习目标: 正确识别用户真实意图（诊断 / 草药 / 养生 / 处方 / 通用）

单线程学习:
# 当前对话中的意图纠偏
用户: "我想开点药"
Agent识别: 处方咨询
用户纠正: "不是，我想查某个草药的功效"
→ 立即学习: "开药" + 无症状描述 → 草药咨询

跨线程学习:
# 从历史案例中提取意图识别规则
案例1: "开药" + 无症状 → 草药咨询
案例2: "开点药" + 无症状 → 草药咨询
案例3: "配点药" + 无症状 → 草药咨询
→ 提取规则: "当用户说'开/配药'但未描述症状时，优先判断为草药咨询"

学习内容:
- 意图识别规则
- 意图消歧策略
- 上下文理解规则

3.2
子图学习维度

3.2
.1
工具选择准确率

学习目标: 在正确的时机选择正确的工具

单线程学习:
# 当前对话中的工具选择纠偏
症状: ["怕冷"]
Agent: 调用辨证工具
结果: 信息不足，无法辨证
→ 立即学习: 症状数量 < 5
时，不应调用辨证工具

跨线程学习:
# 从历史案例中提取工具选择规则
案例1: 症状3个 → 过早辨证 → 失败
案例2: 症状4个 → 过早辨证 → 失败
案例3: 症状6个 → 辨证成功
→ 提取规则: "至少收集5个症状（含舌脉）后才能调用辨证工具"

学习内容:
- 工具调用时机规则
- 工具选择决策树
- 信息完整性检查清单

3.2
.2
症状理解准确率

学习目标: 正确理解用户描述的症状

单线程学习:
# 当前对话中的症状理解纠偏
用户: "我怕冷"
Agent理解: 发冷（恶寒）
用户纠正: "不是发冷，是平时就怕冷"
→ 立即学习: "怕冷"
需要区分
"恶寒（外感）"
vs
"畏寒（阳虚）"

跨线程学习:
# 从历史案例中提取症状理解规则
高频误解: "怕冷"
被误判为
"恶寒"(15
次)
→ 提取规则: "遇到'怕冷'，必须追问：是最近突然怕冷（外感），还是一直怕冷（阳虚）"

学习内容:
- 症状消歧规则
- 症状追问策略
- 症状语义映射表

3.2
.3
辨证推理准确率（最核心）

学习目标: 正确辨别证型

单线程学习:
# 当前对话中的辨证纠偏
症状: 怕冷、腰膝酸软、夜尿频多、舌淡胖
Agent辨证: 脾阳虚
用户反馈: 不对，应该是肾阳虚
→ 立即学习: "腰膝酸软"
是肾的定位症状，不应误判为脾阳虚

跨线程学习:
# 从历史案例中提取辨证鉴别规则
高频错误: "肾阳虚 vs 脾阳虚混淆"(15
次)
共同点: 都有畏寒、疲劳
鉴别点:
- 肾阳虚: 腰膝酸软、夜尿频多
- 脾阳虚: 食少便溏、腹胀
→ 提取规则: "肾阳虚必有腰膝症状或夜尿，脾阳虚必有消化症状，据此区分"

学习内容:
- 证型鉴别规则
- 辨证思路模板
- 常见误诊模式库
- 关键鉴别点清单

3.2
.4
追问策略准确率

学习目标: 高效收集诊断所需信息

单线程学习:
# 当前对话中的追问优化
追问历史: [主诉, 寒热, 其他症状, 寒热(重复), 汗出, 舌脉]
用户反馈: "你第3轮和第4轮都问了寒热"
→ 立即学习: 避免重复追问，需要记住已问过的内容

跨线程学习:
# 从成功案例中提取高效追问路径
肾阳虚成功案例(10
个):
- 平均轮数: 5
轮
- 最优路径: [主诉 → 寒热 → 腰膝 → 夜尿 → 舌脉 → 辨证]

脾阳虚成功案例(8
个):
- 平均轮数: 5
轮
- 最优路径: [主诉 → 寒热 → 消化 → 大便 → 舌脉 → 辨证]

→ 提取策略: 按证型建立最优追问路径库

学习内容:
- 最优追问序列（按证型）
- 追问效率评估规则
- 信息收集完整性检查清单

class MainGraphLearningTriggers:
    """主图学习触发器"""

    def trigger_on_intent_correction(self, state):
        """用户纠正意图时立即触发"""
        if state.get("user_corrected_intent"):
            return True

    def trigger_on_route_mismatch(self, state):
        """路由与用户期望不符时触发"""
        if state.get("route_mismatch_detected"):
            return True


4.3
.2
子图触发条件


class SubgraphLearningTriggers:
    """子图学习触发器"""

    # 1. 工具选择错误时触发
    def trigger_on_tool_error(self, state):
        """工具调用失败或返回错误时"""
        if state.get("tool_error"):
            return True

    # 2. 用户纠正时立即触发
    def trigger_on_user_correction(self, state):
        """用户纠正诊断、症状理解等"""
        if state.get("user_corrected"):
            return True

    # 3. 每N轮追问后触发反思
    def trigger_on_inquiry_rounds(self, state):
        """每5轮追问后反思一次"""
        rounds = state.get("inquiry_rounds", 0)
        if rounds > 0 and rounds % 5 == 0:
            return True

    # 4. 信息收集停滞时触发
    def trigger_on_inquiry_stagnation(self, state):
        """连续3轮追问但信息增量很少"""
        recent_info_gain = state.get("recent_info_gain", [])
        if len(recent_info_gain) >= 3 and sum(recent_info_gain[-3:]) < 0.3:
            return True

    # 5. 诊断完成后触发
    def trigger_on_diagnosis_completion(self, state):
        """诊断完成后反思整个过程"""
        if state.get("diagnosis_completed"):
            return True

def retrieve_similar_complex_cases(
        query_symptoms: List[str],
        query_embedding: np.ndarray,
        top_k: int = 5,
        min_complexity: float = 7.0
) -> List[ComplexCasePattern]:
    """检索相似的疑难案例"""

    # 向量检索（语义相似）
    similar_ids = vector_store.search(
        query_vector=query_embedding,
        top_k=top_k,
        filter={"complexity": {"$gte": min_complexity}}
    )

    # 从数据库加载详细信息
    cases = db.query(ComplexCaseLibrary).filter(
        ComplexCaseLibrary.id.in_(similar_ids)
    ).all()

    return cases

def get_discriminating_rule(
        syndrome_a: str,
        syndrome_b: str
) -> Optional[DiscriminatingRule]:
    """获取两个证型的鉴别规则"""

    rule = db.query(DiscriminatingRule).filter(
        or_(
            and_(
                DiscriminatingRule.confused_syndromes[0] == syndrome_a,
                DiscriminatingRule.confused_syndromes[1] == syndrome_b
            ),
            and_(
                DiscriminatingRule.confused_syndromes[0] == syndrome_b,
                DiscriminatingRule.confused_syndromes[1] == syndrome_a
            )
        )
    ).first()

    return rule

class CrossThreadLearningTriggers:
    """跨线程学习触发器"""

    def should_persist_to_cross_thread(
            self,
            session: LearningSession
    ) -> bool:
        """判断是否应该将本次会话持久化到跨线程知识库"""

        # 条件1: 疑难杂症（复杂度高）
        if session.complexity_score >= 7:
            return True

        # 条件2: 对话轮数超过阈值
        if session.interaction_rounds > 8:
            return True

        # 条件3: 用户纠正次数多
        if session.total_corrections >= 3:
            return True

        # 条件4: 最终成功且用户满意度高（正面案例）
        if session.diagnosis_success and session.user_satisfaction >= 4.5:
            return True

        # 条件5: 失败案例（需要学习避免）
        if not session.diagnosis_success:
            return True

        # 条件6: 罕见证型（稀有案例）
        if session.syndrome_type in RARE_SYNDROMES:
            return True

        return False

    def should_aggregate_knowledge(self) -> bool:
        """判断是否应该聚合跨线程知识"""

        # 每积累50个新案例，触发一次聚合
        if self.pending_cases_count >= 50:
            return True

        # 每24小时触发一次定期聚合
        if self.hours_since_last_aggregation >= 24:
            return True

        # 某个错误模式频繁出现（>10次）
        for pattern, freq in self.error_patterns.items():
            if freq >= 10:
                return True

        return False

async def aggregate_cross_thread_knowledge():
    """聚合跨线程知识"""

    # 1. 提取疑难案例
    complex_cases = extract_complex_cases(
        min_complexity=7.0,
        min_rounds=8
    )
    for case in complex_cases:
        save_to_complex_case_library(case)

    # 2. 提取高效策略
    successful_cases = extract_successful_cases(
        min_satisfaction=4.5,
        max_rounds=6
    )
    effective_strategies = extract_effective_strategies(successful_cases)
    for strategy in effective_strategies:
        save_to_strategy_library(strategy)

    # 3. 提取辨证鉴别规则
    diagnosis_errors = extract_diagnosis_errors(
        min_frequency=5
    )
    discriminating_rules = await extract_discriminating_rules(diagnosis_errors)
    for rule in discriminating_rules:
        save_to_discriminating_rules(rule)

    # 4. 提取误诊模式
    error_patterns = identify_error_patterns(
        min_occurrence=5
    )
    for pattern in error_patterns:
        save_to_misdiagnosis_patterns(pattern)

    # 5. 生成聚合报告
    report = generate_aggregation_report()
    return report
